hello-device-api

HTTP proxy for the Hello Device catalogue. Zero framework, Node built-ins only. One file, one process.

Run local

node src/server.js  # listens on :8787
curl http://localhost:8787/capabilities

Docker

docker compose up -d

Endpoints

Method + PathPurpose
GET /healthLiveness check
GET /capabilitiesFull catalogue (metadata)
GET /groupsThe seven categories
GET /check/:nameAvailability for one capability
GET /test/:nameLive server-side test (browser-only capabilities return a note)
GET /reportGrouped catalogue + summary

Browser-only vs server-testable

Most of the ~45 Web APIs Hello Device tracks only exist in a real browser tab. For those, this API returns metadata + available:null with a note pointing to the SDK. For wasm, crypto, and fetch we actually run the test server-side.
